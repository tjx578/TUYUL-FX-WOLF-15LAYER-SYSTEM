# Candlestick Reference

**Type:** Knowledge reference

## Purpose

This file preserves candlestick concepts as reading aids for analysts and reviewers.

## Usage rule

Candlestick language may help explain a setup, but production entry logic must remain grounded in explicit, testable system rules rather than discretionary pattern names alone.

## Examples of valid human use

- operator training
- review commentary
- post-trade explanation
- mapping visual structure to documented analytical layers
