# Trading Dashhh Legacy UI

This directory reserves the quarantine location for older `Trading Dashhh` style materials.

## Why legacy

These UI concepts are intentionally treated as non-canonical until they are tied to:

- backend contracts
- governance-aware controls
- resolver and lock state
- read-only or explicitly approved operator actions

## Migration rule

Any future dashboard redesign should be reintroduced through `architecture/` plus actual frontend/backend implementation artifacts, not revived directly from legacy UI notes.
